01 · Overview

What This Project Does

This is a personal deep-dive into the OTT content landscape. The dashboard covers two parallel analytical tracks: platform content libraries (what each streaming service offers, by genre, rating, type, and year) and global box office performance (Bollywood vs Hollywood revenue, budgets, scores, and geographic distribution).

Built entirely as a self-initiated project, it demonstrates end-to-end Power BI skills — from raw multi-source data ingestion and schema normalisation, through DAX calculations and relationship modelling, to a polished 10-page interactive dashboard with animated visuals and custom branding per platform.

🎯 Motivation

Personal interest in the streaming wars and box office economics — wanted to visualise how different platforms compare on content volume, genre diversity, and audience ratings.

📦 Data Sources

Six publicly available datasets: Netflix, Amazon Prime, Disney+ Hotstar, Apple TV+ catalogues, plus curated Bollywood and Hollywood box office records.

📊 Output

Ten branded report pages with platform-specific deep-dives, a cross-platform movies overview, separate Hollywood and Bollywood revenue analysis, and scrolling title tickers.

05 · Data Transformation

Power Query ETL Pipeline

Each of the six source files required different transformation work before the data model could be built. Key steps applied across the project:

01

Schema Standardisation

Column names and data types were standardised across all six files — enforcing consistent date formats (Netflix used "24-09-2021" vs Amazon's "March 30, 2021"), numeric types for revenue/budget/runtime, and string types for categorical fields.

02

Netflix Schema Normalisation (Multi-value Columns)

Netflix's cast, director, country, and listed_in (genres) columns each contain comma-separated multiple values. These were split and expanded into separate normalised tables (Netflix Cast, Director, Country, Genre) linked back to Netflix Main via title/show_id.

03

Bollywood Genre Normalisation

Similar to Netflix, the Bollywood genre column was split into a separate Bollywood Genre dimension table to allow genre-level slicer filtering on the Bollywood page.

04

Date Field Extraction (Netflix)

Extracted year, month, and day components from Netflix's date_added field to build the Date Hierarchy used for year-based slicing and the ribbon chart's time axis.

05

Null & Missing Value Handling

Handled missing director, cast, country, and rating values across all platform files — replacing nulls with placeholder values where required for KPI card display logic, and filtering them out where required for clean chart rendering.

06

Revenue / Budget Cleaning (Movies)

Hollywood and Bollywood financial columns required type conversion from string to whole number, removal of currency symbols, and handling of rows with zero or null budget values that would distort budget distribution visuals.

07

Combined Movies View (Final_Imbd)

Appended the cleaned Bollywood and Hollywood tables into a single Final_Imbd table used exclusively on the Movies Overview page — enabling cross-cinema revenue, genre, language, and country analysis in a single report page.

08

Apple TV+ Data Enrichment

Apple TV+'s dataset came with IMDb scores, TMDb popularity, and runtime already included — these were cleaned and retained as additional analytical dimensions not available in the other platform datasets.

06 · Custom Visuals

3D rotating globe for geographic visualisation — used to plot content production countries across platforms in an immersive, cinematic way.

3D bubble/bar chart visual — used on the Hotstar page to represent genre distribution in three-dimensional space, adding visual depth to the exploration.

Live scrolling ticker — used on the Netflix page to display a continuously scrolling list of titles with release year and user ratings, mimicking a real content feed.

Title word cloud — deployed on both Hotstar and Apple TV+ pages to provide an at-a-glance feel for the content catalogue by surfacing frequently occurring title words.

Lollipop / PL Chart

Custom lollipop chart used on the Hotstar page for a cleaner alternative to a standard bar chart when comparing counts across genre or age rating categories.

07 · Tools & Methods

Technical Stack

Power BI Desktop

Primary tool for all 10 report pages — authoring, visual layout, branding, and interactive filtering with action button navigation.

Power Query (M Language)

Full ETL pipeline: schema normalisation, multi-value column splitting, date extraction, null handling, append queries.

DAX

Count, sum, and filter-based calculated measures for KPI cards, gauges, and box office financial aggregations.

Schema Normalisation

Netflix multi-value columns split into five linked tables — practical application of relational design for handling denormalised source data.

AppSource Custom Visuals

Five custom visuals integrated including Globe Map, 3D BI, Scrolling Ticker, Word Cloud, and Lollipop chart — extending Power BI's default visual library.

Cross-domain Data Modelling

Two independent analytical domains (OTT platforms + Movie box office) managed in a single data model — with append queries creating unified cross-domain views.
